================================================================================
THE PROBLEM
================================================================================

Coverage reports show ~19.23%, but actual measured coverage is 5.21%.
The CI gate (25%) is failing because the system is measuring an inconsistent
set of modules, not because tests are failing.

ROOT CAUSE: pyproject.toml contains 15 non-existent package-name --cov paths
that were never removed when code was refactored into suite-* directories.

This causes fragmented measurement where some modules are counted twice (or
inconsistently) based on how coverage.py resolves overlapping paths.

================================================================================
KEY FINDINGS
================================================================================

Finding 1: Configuration Paths Mismatch
  - 15 out of 30 configured --cov paths do NOT exist on filesystem (50%)
  - Examples: --cov=core, --cov=risk, --cov=cli, --cov=automation (dead code)
  - These were legacy package names from before suite-core/, suite-api/, etc.

Finding 2: Massive Uncovered Modules
  - 359 out of 448 modules (80.1%) have 0% coverage
  - Critical entry points untested:
    * suite-core/core/cli.py (2,459 LOC, 0% coverage)
    * suite-api/apps/api/app.py (1,275 LOC, 0% coverage) [FastAPI entry point]
    * 35 routers across suite-api (13,456 LOC, mostly 0% coverage)
    * Risk/evidence modules (7,631 LOC, entirely 0% coverage)

Finding 3: True Coverage Metrics
  - Total statements measured: 67,261
  - Total statements covered: 3,503
  - Actual coverage: 5.21%
  - Reported coverage: 19.23% (artifact of overlapping path aggregation)

Finding 4: Root Cause Mechanism
  When multiple --cov paths overlap or don't resolve:
  - coverage.py may count statements twice (inflating reported percentage)
  - Or skip unmeasured paths silently (understating coverage)
  - This creates the 19.23% vs. 5.21% gap (neither is consistently used)

Finding 5: The Architecture Shift
  - Old structure (2024): core/, api/, risk/ at repo root
  - New structure (2025): suite-core/core, suite-api/apps/api, etc.
  - sitecustomize.py added to support: "from core import X" (backward compatible)
  - pyproject.toml was updated to add NEW paths but NEVER removed OLD paths
