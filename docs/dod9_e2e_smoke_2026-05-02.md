ship in code; the routes named in the DoD-9 spec do not exist in `App.tsx`.

- Imported as: `MissionControlComplianceDashboard` (App.tsx:31)
- Mounted at: **nowhere**
- Closest existing route: `/mission-control/compliance` → `<Navigate to="/compliance" replace />` (App.tsx ~line 700s)
- DoD-5 content ("CTEM Cycles" tab, line 811 of the component) **does exist in the component file**, so the work landed; only the route mount is missing.
- Owner-to-page directive: **frontend-craftsman** — add `<Route path="/mission-control/compliance-dashboard" element={<MissionControlComplianceDashboard />} />` near App.tsx:860.

### FAIL #2 — `/findings/explorer`
- Component: `suite-ui/aldeci-ui-new/src/pages/findings/FindingsExplorer.tsx`
- Mounted at: `/findings` (App.tsx:860) — NOT at `/findings/explorer`
- DoD-8 content does exist in the component (line 1392 has "Related findings" comment).
- Owner-to-page directive: either add `<Route path="/findings/explorer" element={<FindingsExplorer />} />` alongside line 860, OR update DoD-9 spec to canonical `/findings`. Either fix is one line.

### FAIL #3 — `/dashboards/browser-security`
