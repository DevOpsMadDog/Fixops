# Feed Importer API Reference — 2026-04-27

All endpoints require an API key passed as `Authorization: Bearer <key>` (via `api_key_auth` dependency).
All paths are relative to the ALDECI API gateway (e.g. `https://api.aldeci.internal`).
Stub status: every router listed below is **wired-real** — no 501 stubs remain.

## 1. CISA KEV — Known Exploited Vulnerabilities

**Router:** `suite-api/apps/api/vulnerability_correlation_router.py` + `suite-feeds/feeds/cisa_kev/importer.py`
**Source URL:** `https://www.cisa.gov/sites/default/files/feeds/known_exploited_vulnerabilities.json`
**License:** CC0-1.0 (US Government work — public domain)
**Refresh interval:** Daily (86 400 s)
**Feed registry ID:** `cisa_kev`

### POST /api/v1/feeds/registry/cisa_kev/refresh

Trigger a CISA KEV import via the feed registry (preferred entry point).

**Request body:** none

**Response `200`:**

```json
{
  "feed_id": "cisa_kev",
  "status": "ok",
  "imported_at": "2026-04-27T08:00:00+00:00",
  "entry_count": 1123,
  "result": {
    "imported": 12,
    "updated": 0,
    "skipped": 1111,
    "source_count": 1123
  },
  "error": null
}
```

**Response `502`:** upstream CISA endpoint unreachable.

**Example curl:**

```bash
curl -s -X POST \
  -H "Authorization: Bearer $API_KEY" \
  https://api.aldeci.internal/api/v1/feeds/registry/cisa_kev/refresh | jq .
```

## 2. MITRE ATT&CK — Technique Import and Query

**Router:** `suite-api/apps/api/threat_vector_analysis_router.py`
**Source URL:** `https://raw.githubusercontent.com/mitre/cti/master/enterprise-attack/enterprise-attack.json` (STIX 2.1)
**License:** Apache-2.0
**Refresh interval:** Weekly (604 800 s)
**Feed registry ID:** `mitre_attack`

### POST /api/v1/threat-vectors/import-mitre

Download the MITRE ATT&CK Enterprise STIX 2.1 bundle (~10 MB) and upsert all attack-pattern objects into `data/mitre_attack.db`.

**Query params:**

| Param | Type | Required | Description |
|---|---|---|---|
| `org_id` | string | yes | Organization identifier |

**Request body:** none

**Response `200`:**

```json
{
  "org_id": "acme",
  "status": "ok",
  "techniques": 193,
  "subtechniques": 411,
  "tactics": 14,
  "platforms": ["Windows", "Linux", "macOS", "Cloud", "Network"]
}
```

**Response `500`:** STIX bundle download or parse failure.

**Example curl:**

```bash
curl -s -X POST \
  -H "Authorization: Bearer $API_KEY" \
  "https://api.aldeci.internal/api/v1/threat-vectors/import-mitre?org_id=acme" | jq .
```

### GET /api/v1/threat-vectors/mitre-techniques

List imported ATT&CK techniques from the local DB, optionally filtered by tactic.

**Query params:**

| Param | Type | Default | Description |
|---|---|---|---|
| `tactic` | string | null | Filter by tactic slug, e.g. `initial-access`, `execution`, `persistence` |

**Response `200`:**

```json
{
  "techniques": [
    {
      "id": "T1190",
      "name": "Exploit Public-Facing Application",
      "tactic": "initial-access",
      "platforms": ["Linux", "Windows", "macOS", "Network"],
      "description": "Adversaries may attempt to exploit...",
      "url": "https://attack.mitre.org/techniques/T1190"
    }
  ],
  "total": 193
}
```

If no techniques have been imported yet the response is `200` with `"total": 0` and a `"hint"` field pointing to the import endpoint.

**Example curl:**

```bash
curl -s \
  -H "Authorization: Bearer $API_KEY" \
  "https://api.aldeci.internal/api/v1/threat-vectors/mitre-techniques?tactic=initial-access" | jq .
```

## 5. CIS Benchmark — XCCDF Controls

**Router:** `suite-api/apps/api/security_posture_benchmarking_router.py`
**Source URL:** Public SCAP-Repository mirror (CIS Controls v8 XCCDF); admin may supply a local file path for gated downloads
**License:** CIS Terms of Use (redistributable for non-commercial use; commercial use requires CIS SecureSuite membership)
**Refresh interval:** On new CIS benchmark release (manual trigger)

### POST /api/v1/posture-benchmarking/import-cis

Import CIS Benchmark XCCDF controls into the local catalog.

**Request body (JSON, optional):**

```json
{
  "file_path": "/opt/cis/cis-ubuntu22-xccdf.xml",
  "url": null,
  "idempotent": true
}
```

| Field | Type | Default | Description |
|---|---|---|---|
| `file_path` | string | null | Local XCCDF file path (admin-uploaded fallback when source is gated) |
| `url` | string | null | Override the CIS XCCDF source URL |
| `idempotent` | bool | true | Skip controls already present in DB |

Source resolution order: `file_path` > `url` > default public SCAP-Repository mirror.

**Query params:**

| Param | Type | Default | Description |
|---|---|---|---|
| `org_id` | string | `default` | Organization identifier |

**Response `200`:**

```json
{
  "imported": 171,
  "skipped": 0,
  "benchmark_id": "cis-ubuntu-22.04-l1",
  "title": "CIS Ubuntu Linux 22.04 LTS Benchmark",
  "version": "1.0.0"
}
```

**Response `502`:** Source URL unreachable — includes `remediation` field with instructions to supply `file_path`.
**Response `422`:** Malformed or non-XCCDF document.

**Example curl:**

```bash
curl -s -X POST \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"idempotent": true}' \
  "https://api.aldeci.internal/api/v1/posture-benchmarking/import-cis?org_id=acme" | jq .
```

### GET /api/v1/posture-benchmarking/cis-controls
