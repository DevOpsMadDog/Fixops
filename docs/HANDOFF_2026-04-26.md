## Critical operating constraints (NON-NEGOTIABLE for next agent)

1. **NO MOCKS RULE.** See top of CLAUDE.md. Every UI task ends with: navigate via Playwright → screenshot → DOM-inspect for mock signatures → confirm `/api/v1/...` call fires on mount → if mock found, fix the API integration not the UI. **TypeScript compiling is NOT enough.**

2. **REAL CUSTOMERS, NOT SEEDED DATA.** "Test with real apps" = onboard via the actual customer flow (org → connector → sync → pipeline). Direct DB writes = same as a mock.
