# Session Retrospective — 2026-04-26

## What Worked (Top 5 — Measurable)

1. **Parallel agent dispatch** — 6-12 agents in flight simultaneously, ~140 commits shipped in a single session. No sequential bottleneck. Tasks that could run independently were never serialized.

2. **Salvage-on-stall pattern** — When agents stalled mid-commit on heredoc bugs, CTO pulled code off disk and committed manually rather than re-dispatching from scratch. Zero lost work across 5+ stall incidents.

3. **Memory.md auto-feedback loop** — `feedback_no_tails_no_asks`, `feedback_no_screens`, `no-mocks-routers` rules written mid-session; behavior corrected immediately for all subsequent dispatches. Feedback persists across sessions without re-teaching.

4. **Junior-worker for templated tasks** — Iron Bank Dockerfile prep, Tenable AI Exposure tab placement, sentence-transformers install all routed to junior/cheap models. Crisp output, no Opus burn on mechanical work.

5. **NO MOCKS / REAL CUSTOMERS rule** — Real MiniLM embeddings, real fleet scans, real GHSA import produced verifiable evidence. DPO pairs grew 2 → 1823 (911x). TrustGraph coverage 24.4% → 38.4%. Numbers are real, not seeded.

---

## What Didn't Work (Top 5)

1. **Heredoc git-commit bug** — At least 5 agents stalled on multi-line heredoc commit messages. Lost wall-clock time on each incident before switching to `git commit -F /tmp/msg.txt`.

2. **CTO summary-then-pause pattern** — User raised this 6+ times during the session. Agent defaulted to summarizing completed work and waiting for acknowledgment. Fixed only after `feedback_no_tails_no_asks` memory was written.

3. **ruflo init overwrote CLAUDE.md** — A real safety incident. ruflo's `init` command silently clobbered the project operating manual. Recovered from git, but the risk is non-zero on every `init` call.

4. **Visualizer measured by file count, not blast radius** — Early AQUA color iteration reported % based on file presence; needed a second pass to reflect actual call-graph coverage. Misleading metric surfaced before honest one.

5. **Opus dispatched on tasks that should be junior-worker** — Mechanical tasks (templated scaffolding, file moves, boilerplate inserts) burned Opus budget unnecessarily. Memory `feedback_plan_strategy.md` exists but wasn't pre-loaded at session start.

---

## Process Improvements for Next Session (Top 3 — Actionable)

1. **Pre-emptive heredoc avoidance in all agent prompts** — Every dispatch template that includes a `git commit` step must use `git commit -F /tmp/msg.txt` form. Add this as a hard rule to CLAUDE.md agent prompt section.

2. **Pre-load top 3 memories at session start** — Before first dispatch, explicitly read and acknowledge: `feedback_no_tails_no_asks`, `feedback_no_mocks_routers`, `feedback_no_screens`. Apply them to the first prompt template, not the sixth.

3. **Junior-worker routing checklist** — Before dispatching to Opus, ask: "Is this templated, mechanical, or under 50 LOC?" If yes, route to junior/cheap model. Add routing decision to the dispatch log line.
